The annual CppCon conference landed this year from September 25th to September 29th. Since then, I try to find some time to watch videos of the presentations.

One video attracted my intention. In this video, Kate Gregory explains why you should 'Stop teaching C' as a pre-requisite to a C++ course. I can not less agree with her on this point...

Unfortunately, C++ is known as a hard to learn language, and this reputation is in fact true if you learn it the hard way. So if you have to teach C++, start by presenting higher level C++. When your students have understood high-level C++ basics, learn them how to do the low level things.

For example, here is a list a things you should avoid in a 'day 1' beginner course :

  • char* strings : If you use C-style strings, you'll have to explain what pointers are, and present the tricky str* functions. Just use std::string as if it were a standard type.
  • printf style formatting : Just use iostreams. The performance is not so important when beginning C++.
  • C-style arrays : As with C-style strings, this bring pointers on the table far too soon. Just use std::vectors.
In conclusion, just avoid using low level C-style features if you can do it using high-level C++ features. Kate went far further, so if you're interested, go watch the CppCon video, or even listen to the CppCast episode !
